## Versioning

You might see some of my version numbers and be like "what could `1.0.3C-R2` mean. I'll go through that.

- The `1` is the major version number. This is exactly what you think it is
- The `0` is the minor version number. This is used for more minor updates, as you can assume.
- The `3` is the update version number. This is used for small improvements or bug fixes.
- The `C` is the patch version number. This is used for bug fixes.
- The `R2` means `release-2`. A "release" is an iteration. You'll rarely see this one, as this is primarily used for re-releasing the same beta version. It's never used in release builds.

In restricted environments (where my version numbers have to follow a specific pattern), you might see the patch number and optional release number shown as a separate number (like `1.0.3.3.2` in this case). In even more restricted environments (like pub.dev), there might only be three numbers, where the third is actually the patch number.
